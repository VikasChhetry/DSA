Definition of SQLite
SQLite is a C-language library that implements a small, fast, self-contained, high-reliability, full-featured SQL database engine. It is the most widely deployed database engine in the world. Unlike other databases like MySQL or Oracle, SQLite does not require a separate server — it stores the entire database in a single file on disk and can be embedded directly into any application.
Key exam point: SQLite is serverless and zero-configuration — no admin setup needed.

Features of SQLite (exam favourite)
1
Serverless
No separate server process needed. The database engine runs in the same process as your application.
2
Zero-configuration
No installation, no setup, no administration required. Just include the library and use it.
3
Single cross-platform file
The entire database (tables, indexes, data) lives in one .db file that works on any OS.
4

Less than 400KB fully configured, or less than 250KB with optional features omitted.
5
Self-contained
No external dependencies — the library itself is all you need.
6
ACID-compliant transactions
Supports Atomicity, Consistency, Isolation, Durability — safe access from multiple threads/processes.
7
Supports SQL92 standard
Supports most SQL2 query language features (CREATE, SELECT, INSERT, UPDATE, DELETE, etc.)
8
Written in ANSI-C
Simple and easy-to-use API, portable across all systems.
9
Cross-platform
Available on UNIX (Linux, macOS, Android, iOS) and Windows (Win32, WinCE, WinRT).

Import the module
import sqlite3 — Python has sqlite3 built-in, no pip install needed.
2

conn = sqlite3.connect('mydb.db') — creates the file if it does not exist.
3
Create a cursor
cursor = conn.cursor() — cursor is the object that executes SQL queries.
4
Execute a query
cursor.execute('SELECT sqlite_version()')
5
Fetch results
result = cursor.fetchall() — gets all rows returned by the query.
6

import sqlite3 conn = sqlite3.connect('students.db') cursor = conn.cursor() cursor.execute('''CREATE TABLE IF NOT EXISTS student (RollNo INT PRIMARY KEY, Name TEXT, Age INT)''') cursor.execute("INSERT INTO student VALUES (101, 'Aman', 20)") cursor.execute("INSERT INTO student VALUES (102, 'Raman', 21)") conn.commit() cursor.execute("SELECT * FROM student") rows = cursor.fetchall() for row in rows: print(row) conn.close()
Cursor methods — must know
Important cursor methods
execute()
Runs a single SQL statement
cursor.execute("SELECT * FROM t")
executemany()
Runs same SQL for a list of tuples (multiple rows at once)
cursor.executemany("INSERT INTO t VALUES (?,?)", data)
fetchall()
Returns all rows from last SELECT as a list
rows = cursor.fetchall()
fetchone()
Returns just the first row from the result
row = cursor.fetchone()
commit()
Saves changes permanently (on connection, not cursor)
conn.commit()
close()
Closes connection and frees resources
conn.close()
Remember: Use conn.commit() after INSERT/UPDATE/DELETE. Without it, changes are NOT saved to disk.
Using placeholders (safe queries)
Parameterized queries — use ? for safe input
import sqlite3 conn = sqlite3.connect('students.db') cursor = conn.cursor() name = "Aman" age = 20 cursor.execute("INSERT INTO student VALUES (?, ?)", (name, age)) cursor.execute("SELECT * FROM student WHERE Name = ?", (name,)) result = cursor.fetchall() print(result) conn.commit() conn.close()
Using ? prevents SQL injection attacks — always prefer this over string formatting.

Definition of DDL (Data Definition Language)
DDL stands for Data Definition Language. It consists of SQL commands that define and manage the structure (schema) of a database. DDL deals with the skeleton of the database — creating tables, modifying their structure, and deleting them. DDL commands auto-commit — changes cannot be rolled back.
DDL commands — all 6 with examples
C
CREATE — creates a new table/database/view/index
CREATE TABLE student ( RollNo INT PRIMARY KEY, Name TEXT NOT NULL, Age INT, Email TEXT );
D
DROP — deletes the entire table including all data
DROP TABLE student; -- Careful: this removes the table completely!
A
ALTER — modifies existing table structure (add/rename/drop columns)
-- Add a new column ALTER TABLE student ADD COLUMN Phone TEXT; -- In Python using MariaDB: cur.execute("ALTER TABLE contacts ADD COLUMN IF NOT EXISTS (contact_since INT)")
T
TRUNCATE — deletes all rows but keeps the table structure
TRUNCATE TABLE student; -- Table remains, but all rows are deleted
R
RENAME — renames an existing table
ALTER TABLE student RENAME TO learner;
C
COMMENT — adds comments to the data dictionary
COMMENT ON TABLE student IS 'Stores student records';
DDL in Python (mariadb connector)
ALTER TABLE using Python — full code
import mariadb import sys def alter_contacts(cur): try: cur.execute( "ALTER TABLE test.contacts ADD COLUMN IF NOT EXISTS (contact_since INT)" ) except mariadb.Error as e: print(f"Error altering contacts: {e}") conn.close() sys.exit(1) try: conn = mariadb.connect( host="localhost", port=3306, user="root", password="password" ) except mariadb.Error as e: print(f"Error connecting to database: {e}") sys.exit(1) cur = conn.cursor() alter_contacts(cur) conn.close()
Key note: The function must be defined before it is called. Always call conn.close() at the end to free resources.

Definition of DML (Data Manipulation Language)
DML stands for Data Manipulation Language. These are SQL commands used to manipulate (add, change, delete, retrieve) the actual data stored inside tables. Unlike DDL which changes structure, DML changes content. DML commands can be rolled back (undone) if not committed.
INSERT
Adds new rows of data into a table
UPDATE
Modifies existing data in rows
DELETE
Removes rows from a table
SELECT
Reads and retrieves data (also called DQL)
Write all 4 with code in exam
INSERT — adding data
-- Insert a single row INSERT INTO student (RollNo, Name, Age) VALUES (101, 'Aman', 20); -- Insert without column names (must match order) INSERT INTO student VALUES (102, 'Raman', 21); -- Insert multiple rows at once INSERT INTO student VALUES (103, 'Kiran', 22), (104, 'Priya', 20);
UPDATE — modifying data
-- Update a specific row UPDATE student SET Age = 25 WHERE RollNo = 101; -- Update multiple columns UPDATE student SET Age = 22, Name = 'Aman Kumar' WHERE RollNo = 101; -- Update all rows (no WHERE — be careful!) UPDATE student SET Age = 20;
DELETE — removing data
-- Delete a specific row DELETE FROM student WHERE RollNo = 101; -- Delete rows matching condition DELETE FROM student WHERE Age < 18; -- Delete all rows (table still exists) DELETE FROM student;
SELECT — reading data
-- Get all rows and columns SELECT * FROM student; -- Get specific columns SELECT Name, Age FROM student; -- With a condition (WHERE) SELECT * FROM student WHERE Age > 20; -- With AND / OR SELECT * FROM student WHERE Age > 18 AND Name = 'Aman'; -- Sort results SELECT * FROM student ORDER BY Age DESC;
DML in Python — full code example
INSERT, UPDATE, DELETE using Python + mariadb
import mariadb import sys def add_contact(cur, first, last, email): cur.execute( "INSERT INTO test.contacts(first_name, last_name, email) VALUES (?, ?, ?)", (first, last, email) ) def add_multiple(cur, data): cur.executemany( "INSERT INTO test.contacts(first_name, last_name, email) VALUES (?, ?, ?)", data ) try: conn = mariadb.connect( host="localhost", port=3306, user="root", password="password", autocommit=True # Important! Must set to True for auto-save ) except mariadb.Error as e: print(f"Connection error: {e}") sys.exit(1) cur = conn.cursor() add_contact(cur, "Aman", "Sharma", "aman@example.com") contacts = [ ("Raman", "Singh", "raman@example.com"), ("Priya", "Verma", "priya@example.com"), ("Kiran", "Kumar", "kiran@example.com") ] add_multiple(cur, contacts) cur.execute("SELECT * FROM test.contacts") for row in cur.fetchall(): print(row) conn.close()
autocommit=True is required because by default Python DB connectors do NOT auto-save. Without it, your INSERT/UPDATE/DELETE will not be written to disk.
Error handling — important concept
What is error handling in SQLite?
Error handling means writing code that can gracefully handle problems when database operations fail — like connection errors, duplicate keys, or invalid queries. Instead of the program crashing, it catches the error and shows a useful message.
Three ways to access SQLite errors
1
Third parameter to sqlite_open()
If SQLite cannot open the database, it returns false and stores the error message in $sqlite_error (passed by reference).
2
sqlite_last_error()
Returns the error code (a number) for the most recent error. Resets after every query — so check immediately after each operation.
3
sqlite_error_string()
Converts the numeric error code into a human-readable error message.
Error handling code — Python (most important)
try/except for SQLite errors in Python
import sqlite3 try: conn = sqlite3.connect('students.db') cursor = conn.cursor() cursor.execute('''CREATE TABLE IF NOT EXISTS student (RollNo INT PRIMARY KEY, Name TEXT)''') cursor.execute("INSERT INTO student VALUES (101, 'Aman')") cursor.execute("INSERT INTO student VALUES (101, 'Raman')") # Duplicate! conn.commit() except sqlite3.IntegrityError as e: print(f"Integrity Error (e.g. duplicate key): {e}") except sqlite3.OperationalError as e: print(f"Operational Error (e.g. table not found): {e}") except sqlite3.Error as e: print(f"General SQLite Error: {e}") finally: if conn: conn.close() print("Connection closed.")
The finally block always runs — even if an error occurs. Use it to ensure the connection is always closed.
Common SQLite error types
IntegrityError
Primary key violation, UNIQUE constraint failed
OperationalError
Table not found, wrong syntax, disk full
ProgrammingError
Wrong number of bindings, cursor used after close
DatabaseError
Parent class for all database-related errors
sqlite3.Error
Catches all SQLite errors (most general)

What is NoSQL?
NoSQL stands for "Not Only SQL". It refers to a type of database that does not use the traditional relational table model. Instead of storing data in rows and columns, NoSQL databases store data in flexible formats like documents, key-value pairs, graphs, or columns. NoSQL databases are designed for large-scale, distributed, high-performance applications where data structure may vary.
Types — always asked
Four types of NoSQL databases
Document stores
Data stored as JSON/BSON documents. Example: MongoDB, CouchDB
Key-value stores
Simple key → value pairs. Example: Redis, DynamoDB
Column-family stores
Data in columns grouped by family. Example: Cassandra, HBase
Graph databases
Data as nodes and edges for relationships. Example: Neo4j

Definition — write this in exam
What is MongoDB?
MongoDB is an open-source, document-oriented NoSQL database designed to store large amounts of data. Data is stored as BSON (Binary JSON) documents instead of rows. It provides high performance, high availability, and easy scalability. MongoDB is used by Facebook, Nokia, eBay, Adobe, and Google. It supports languages including Python, Java, Node.js, C++, Go, PHP, Ruby, and more.
Features — write all 6 in exam
Features of MongoDB
1
Schema-less database
One collection can hold documents with completely different fields. No need to define a schema before inserting data.
2
Document-oriented
All data is stored as documents (in BSON format — Binary JSON). Each document has a unique _id field (acts as primary key).
3
Indexing
Every field can be indexed (primary and secondary). This makes searching much faster — less time to find data in large collections.
4
Scalability (Sharding)
MongoDB distributes data across multiple servers using sharding. A shard key splits data into chunks and spreads them across servers. New servers can be added to a running database.
5
Replication
MongoDB creates multiple copies of data on different servers (replica sets). If one server fails, data is retrieved from another. Ensures high availability and zero data loss.
6
Aggregation
Allows performing operations on grouped data. Like GROUP BY in SQL. Provides three methods: aggregation pipeline, map-reduce, and single-purpose aggregation.
Max document size = 16MB. Max nesting depth = 100 levels. Data stored as BSON (Binary JSON).

Advantages and disadvantages
Advantages of MongoDB
Schema-less flexibility · No JOIN operations needed · High performance and availability · Horizontal scalability · Supports ACID transactions (from v4.0) · Geospatial support · BSON document format · No SQL injection risk · Easily integrates with Big Data Hadoop · Heterogeneous data storage
Disadvantages of MongoDB
High memory usage for data storage · Document size limited to 16MB · BSON nesting limited to 100 levels · No JOIN support (must denormalize data) · Less mature than traditional RDBMS for complex transactions

Comparison Query Operators
$eq
Equal to — matches exact value
db.Faculty.find({Age:{$eq:32}})
$ne
Not equal to — excludes exact value
db.Faculty.find({Age:{$ne:32}})
$gt
Greater than
db.Faculty.find({Age:{$gt:30}})
$gte
Greater than or equal to
db.Faculty.find({Age:{$gte:30}})
$lt
Less than
db.Faculty.find({Age:{$lt:40}})
$lte
Less than or equal to
db.Faculty.find({Age:{$lte:40}})
$in
Matches any value in the given array
db.Faculty.find({Age:{$in:[30,32,35]}})
$nin
Matches none of the values in array
db.Faculty.find({Age:{$nin:[30,32]}})
-- Sort results: 1 = Ascending, -1 = Descending db.Faculty.find({Age:{$nin:[32,30]}}).sort({Age: 1}) -- ASC db.Faculty.find({Age:{$nin:[32,30]}}).sort({Age: -1}) -- DESC
Logical operators — all 4 with syntax
Logical Query Operators
$and
AND — both conditions must be true
db.Faculty.find({ $and: [{Age:{$gt:30}}, {CITY:"NOIDA"}] })
$or
OR — at least one condition must be true
db.Faculty.find({ $or: [{Age:{$lt:35}}, {CITY:"DELHI"}] })
$not
NOT — reverses the condition (returns what does NOT match)
db.Faculty.find({ Age: {$not: {$eq: 32}} })
$nor
NOR — returns documents that fail ALL conditions
db.Faculty.find({ $nor: [{CITY:"DELHI"}, {Age:{$gt:40}}] })

What is Aggregation in MongoDB?
Aggregation is a way to process multiple documents and return computed results. It works like GROUP BY in SQL — you can calculate total, average, minimum, maximum, and count values. Aggregation works through a pipeline where documents pass through multiple stages, and each stage transforms the data.
-- Syntax db.collectionName.aggregate(pipeline, options) -- Example: total age by department, only for NOIDA city db.Faculty.aggregate([ {$match: {CITY: "NOIDA"}}, {$group: {_id: "$Deptt", totalAge: {$sum: "$Age"}}} ])
All 5 pipeline stages — write with examples
Aggregation Pipeline Stages
1
$match — filter documents (like WHERE in SQL)
db.Faculty.aggregate([ {$match: {CITY: "NOIDA"}} ])
2
$group — group documents and compute values (like GROUP BY)
-- Count documents per city db.Faculty.aggregate([ {$group: {_id: "$CITY", count: {$sum: 1}}} ]) -- Total age by city db.Faculty.aggregate([ {$group: {_id: "$CITY", totalAge: {$sum: "$Age"}}} ]) -- Min and Max age by city db.Faculty.aggregate([ {$group: {_id: "$CITY", minAge: {$min: "$Age"}, maxAge: {$max: "$Age"}}} ])
3
$sort — sort results (1 = asc, -1 = desc)
db.Faculty.aggregate([{$sort: {Age: 1}}]) -- ascending db.Faculty.aggregate([{$sort: {Age: -1}}]) -- descending
4
$project — choose which fields to include/exclude
db.Faculty.aggregate([ {$project: {CITY: 1, Deptt: 1, Age: 1}} -- show only these 3 fields ])
5
$skip — skip first N documents
db.Faculty.aggregate([{$skip: 5}]) -- skip first 5 documents
$group operators — all you need
$group aggregation operators
$sum
Calculates the total / count (use 1 to count documents)
{$sum: "$Age"} or {$sum: 1}
$avg
Calculates the average value of a field
{$avg: "$Age"}
$min
Returns the minimum value of a field
{$min: "$Age"}
$max
Returns the maximum value of a field
{$max: "$Age"}
$first
Returns value from the first document in the group
{$first: "$Name"}
$last
Returns value from the last document in the group
{$last: "$Name"}

Q13 — MEDIUM PRIORITY (5 marks)
Does MongoDB support ACID transactions? Define ACID.
Yes — MongoDB supports multi-document ACID transactions starting from version 4.0.

ACID definition:
A — Atomicity: a transaction is all-or-nothing. Either all operations succeed or none are applied.
C — Consistency: the database remains in a valid state before and after the transaction.
I — Isolation: concurrent transactions do not interfere with each other.
D — Durability: once committed, changes are permanent even if the system crashes.
Q14 — MEDIUM PRIORITY (5 marks)
Explain how to add or delete columns from an existing SQLite table.
Use ALTER TABLE command to add a column:
ALTER TABLE student ADD COLUMN phone TEXT;

SQLite does not support DROP COLUMN directly (older versions). Workaround: create new table without the column, copy data, drop old table, rename new table.

In Python: cursor.execute("ALTER TABLE student ADD COLUMN phone TEXT") then conn.commit().
