Working with Text & Sequences

Computers don't understand text. We convert words → numbers → sequences for the model.

Why Convert Text to Numbers?
Computers only work with numbers, not strings
Each word gets a unique number called an index
Example: "I love AI" → {I=1, love=2, AI=3} → [1, 2, 3]

Real-world uses:
Chatbots (customer support)
Google Translate
Email spam detection
Tokenizer + Padding (Must Know Code)
from tensorflow.keras.preprocessing.text import Tokenizer from tensorflow.keras.preprocessing.sequence import pad_sequences texts = ["I love machine learning", "Deep learning is powerful"] tokenizer = Tokenizer() tokenizer.fit_on_texts(texts) # Learn vocabulary sequences = tokenizer.texts_to_sequences(texts) # Convert to numbers print(tokenizer.word_index) # See word→number mapping padded = pad_sequences(sequences, padding='post') # Make equal length
Tokenizer → Learns all unique words and assigns them numbers
texts_to_sequences → Converts sentence to list of numbers
pad_sequences → Adds zeros to make all sequences equal length (needed for batch training)
padding='post' → Zeros added at the END of shorter sequences
Why padding?
Neural networks need fixed-size inputs. Padding adds zeros to shorter sentences so all are same length.

Advanced RNN — LSTM & GRU

RNNs = Neural networks for sequential data (text, time series). They remember past information.

Problem with Basic RNN
Basic RNNs forget long-term dependencies
This is called the Vanishing Gradient Problem
Solution: Use LSTM or GRU instead
LSTM vs GRU — Comparison
LSTM (Long Short-Term Memory)
3 gates: Input, Forget, Output
Also has a Cell State (conveyor belt)
Good for long sequences
More complex, slower
Introduced by Hochreiter & Schmidhuber (1997)
GRU (Gated Recurrent Unit)
2 gates: Reset Gate, Update Gate
No separate cell state
Simpler design
Faster, often performs well
Introduced as alternative to vanilla RNN
Key Difference (Exam Favourite)
LSTM has 3 gates + cell state | GRU has 2 gates, no cell state | Both solve vanishing gradient problem
LSTM Gates — What Each Does
GateJob
Input GateDecides which new values to add to cell state
Forget GateDecides what old information to throw away
Output GateDecides what to output as hidden state
Cell StateCarries long-term memory like a conveyor belt
GRU Gates — What Each Does
GateJob
Reset GateHow much of previous memory to combine with new input
Update GateHow much of old state to keep vs update

TensorBoard Visualization

TensorBoard = TensorFlow's visualization toolkit. It helps you monitor and understand your model training.

What can TensorBoard show you?
Track loss and accuracy over epochs (training progress)
Visualize the model graph (layers and operations)
View histograms of weights and biases
Project embeddings to lower dimensions
Display images, text, audio data
Profile TensorFlow programs (find bottlenecks)
TensorBoard Setup Code
import datetime from tensorflow.keras.callbacks import TensorBoard # Step 1: Load extension %load_ext tensorboard # Step 2: Set log directory log_dir = "logs/fit/" + datetime.datetime.now().strftime("%Y%m%d-%H%M%S") # Step 3: Create callback tensorboard_callback = TensorBoard(log_dir=log_dir, histogram_freq=1) # Step 4: Pass callback while training model.fit(x, y, epochs=10, callbacks=[tensorboard_callback]) # Step 5: Launch TensorBoard %tensorboard --logdir logs/fit
callbacks=[tensorboard_callback] → TensorBoard monitors training automatically
histogram_freq=1 → Record weight histograms every epoch
log_dir → Folder where all training logs are saved
Exam Tip

Word Vectors (Word2Vec)

Word Vectors = way to represent words as numbers so computers can understand their meaning.

What is Word2Vec?
Unsupervised word embedding method
Learns word representations from context (surrounding words)
Based on the distributional hypothesis: words that appear in similar contexts have similar meanings
Most popular method: Skip-gram model

Skip-gram idea:

Given a word, predict its context (neighbors).
Sentence: "Our company provides smart agriculture solutions..."
Context of "company" = [our, provides]
Context of "AI" = [advanced, deep-learning]

In simple words
"King - Man + Woman = Queen" — Word vectors capture relationships between words mathematically.
Role of Word Vectors in NLP
Encodes words numerically so models can process them
Similar words get similar vectors (cat & kitten are close)
Used in: translation, sentiment analysis, chatbots
In TensorFlow: use Embedding layer → converts word index to dense vector

Queues & Threads (Data Pipeline)
Thread: Unit of execution — loads data while model trains (parallel work)
Queue: FIFO data structure — stores data temporarily for the model
Together they create a continuous data pipeline → no idle waiting

Analogy:

Restaurant: Chef cooks (model trains) while waiters (threads) bring ingredients continuously. Queue = the serving counter where food waits.

Distributed TensorFlow
Train model on multiple CPUs/GPUs/machines simultaneously
Same model copied to multiple devices → each trains on different data part
Results combined at the end
Benefits: Faster training, handles large data, efficient hardware use

MirroredStrategy = mirrors model on all available GPUs
strategy.scope() = build the model inside this scope for distribution

Transfer Learning — Quick Summary
Use a model trained on one task and apply it to a different but related task
Instead of training from scratch, start with pre-trained weights
Example: Use VGG16 (trained on ImageNet) → fine-tune for medical imaging
Benefits: Saves time, needs less data, higher accuracy
In TensorFlow: load pre-trained model → freeze layers → add new layers → retrain
Analogy
Like a doctor who already knows biology — they don't re-learn it, they just learn the new specialty on top.
