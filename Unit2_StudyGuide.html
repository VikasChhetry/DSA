Built-in Functions in R

🔥 Very Likely 10M
💡 Think of built-in functions as "ready-made tools" in R — you don't need to build them yourself. Just call them!

What are Built-in Functions?

Functions that come pre-installed with R — no package needed. They are divided into categories based on what they do.

📐 Category 1: Mathematical Functions

abs(-5) # 5 — removes negative sign sqrt(16) # 4 — square root ceiling(4.2) # 5 — round UP always floor(4.9) # 4 — round DOWN always round(3.567, 2) # 3.57 — round to 2 decimal places exp(1) # 2.718 — e^1 log(100, 10) # 2 — log base 10 of 100

📊 Category 2: Statistical Functions

x <- c(10, 20, 30, 40, 50) mean(x) # 30 — average median(x) # 30 — middle value sd(x) # 15.81 — standard deviation var(x) # 250 — variance range(x) # 10 50 — min and max sum(x) # 150 min(x) # 10 max(x) # 50 summary(x) # Min, Q1, Median, Mean, Q3, Max quantile(x) # 0%, 25%, 50%, 75%, 100%

🔤 Category 3: Character / String Functions

nchar("Hello") # 5 — count characters toupper("hello") # "HELLO" tolower("HELLO") # "hello" paste("Hello", "World") # "Hello World" substr("Hello", 1, 3) # "Hel" — characters 1 to 3 grep("a", c("cat","dog")) # 1 — position where "a" found

🔧 Category 4: Utility / Apply Functions

# sapply — apply function to each column, returns vector sapply(mydata, mean, na.rm=TRUE) # Mean of each column # apply — for matrices (1=row, 2=column) m <- matrix(1:6, nrow=2) apply(m, 1, sum) # Sum of each row apply(m, 2, sum) # Sum of each column # lapply — returns a list lapply(c(1,4,9), sqrt) # [[1]] 1 [[2]] 2 [[3]] 3

📈 Category 5: Visualization Functions (Base R)

hist(x) # Histogram plot(x, y) # Scatter plot boxplot(x) # Box plot — shows quartiles & outliers barplot(table(x)) # Bar chart pie(table(x)) # Pie chart
⚠️ Exam Tip

For 10 marks: Write at least 2–3 functions per category with working code. Always mention mean(), median(), sd(), summary() and sapply() — they appear most in questions.

Data Structures in R

⚡ Likely 6/10M
💡 Data structures = different "containers" to store data. Like how you use a box for shoes, a rack for books — R has different containers for different data needs.

1️⃣ Vector — Most Basic

A 1D collection of same-type elements. Think of it like a single row in Excel.

v1 <- c(10, 20, 30) # numeric vector v2 <- c("A", "B", "C") # character vector v3 <- 1:10 # sequence 1,2,3,...,10 v4 <- seq(1, 10, by=2) # 1 3 5 7 9 # Accessing elements v1[2] # 20 — second element v1[c(TRUE,FALSE,TRUE)] # 10 30 — logical indexing v1[-1] # 20 30 — exclude first

2️⃣ List — Mixed Type Container

Like a bag that can hold anything — numbers, strings, vectors, even other lists!

mylist <- list("Alice", 25, TRUE, c(1,2,3)) mylist[[1]] # "Alice" mylist[[4]] # 1 2 3

3️⃣ Matrix — 2D Same-Type Table

m <- matrix(1:6, nrow=2, ncol=3) # [,1] [,2] [,3] # [1,] 1 3 5 # [2,] 2 4 6 m[1,2] # row 1, col 2 = 3

4️⃣ Data Frame — Most Important! (Like a Spreadsheet)

2D table where each column can have a different data type. This is what you use for real datasets!

df <- data.frame( Name = c("Alice", "Bob", "Charlie"), Age = c(21, 22, 23), Fee = c(50000, 60000, 55000) ) df$Name # Access column by name df[1,] # Access first row df[df$Age > 21,] # Filter rows where Age > 21

5️⃣ Factor — For Categories

gender <- factor(c("Male", "Female", "Male")) levels(gender) # "Female" "Male" — unique categories
✅ Remember for Exam

Vector = 1D same type | List = 1D mixed type | Matrix = 2D same type | Data Frame = 2D mixed type | Array = nD same type | Factor = categories

dplyr Package — Data Manipulation

🔥 Very Likely 10M
💡 dplyr = a toolkit for working with data frames. It's like Excel but with code. The name means "data frame pliers" — a tool to manipulate data frames.

Installation

install.packages("dplyr") library("dplyr")

📌 Sample Data (use this in all examples)

df <- data.frame( Name = c("Alice", "Bob", "Charlie", "Diana"), Branch = c("CSE", "AIML", "CSE", "AIML"), Salary = c(30000, 45000, 60000, 35000), Age = c(22, 25, 28, 23) )

1. filter() — Select Rows by Condition

Like "WHERE" in SQL — keeps only the rows that match your condition.

filter(df, Salary > 40000) # Bob, Charlie filter(df, Branch == "CSE") # Alice, Charlie filter(df, Branch == "CSE" & Age < 25) # Alice only

2. arrange() — Sort Rows

Like "ORDER BY" in SQL — reorders rows based on a column.

arrange(df, Salary) # Ascending (low to high) arrange(df, desc(Salary)) # Descending (high to low) arrange(df, Branch, Age) # Sort by Branch, then Age

3. select() — Pick Columns

Like "SELECT col1, col2" in SQL — shows only specific columns.

select(df, Name, Salary) # Only Name and Salary columns select(df, -Age) # All columns EXCEPT Age

4. mutate() — Add New Columns ⭐ Most Asked!

Creates new columns WITHOUT removing old ones. Like adding a formula column in Excel.

mutate(df, Tax = Salary * 0.1) # Adds Tax column mutate(df, Net = Salary - Salary*0.1) # Adds Net salary mutate(df, Bonus = ifelse(Branch=="CSE", 5000, 3000))

5. transmute() — Add Columns but DROP Old Ones

transmute(df, Net = Salary * 0.9) # Returns ONLY Net column

6. rename() — Rename a Column

rename(df, Department = Branch) # "Branch" becomes "Department" # Syntax: rename(df, newName = oldName)

7. summarize() — Aggregate Data

summarize(df, AvgSalary = mean(Salary), Total = sum(Salary)) # Returns: AvgSalary=42500, Total=170000

8. distinct() — Remove Duplicate Rows

distinct(df, Branch) # Unique branches: CSE, AIML distinct(df, Branch, .keep_all=TRUE) # Keep all cols
⚠️ Exam Tip — DataFrame Question

If asked to create a Student DataFrame and apply operations: create it with data.frame(), then use filter() for conditions, rename() for renaming, mutate() to add columns. Write ALL the code step by step.

# EXPECTED QUESTION SOLUTION — Student DataFrame student <- data.frame( RollNo = c(101, 102, 103), Name = c("A", "B", "C"), Age = c(20, 21, 22), Course = c("AIML", "CSE", "AIML"), Fee = c(55000, 45000, 60000) ) # a) Rename Course to Branch student <- rename(student, Branch = Course) # b) Students paying > 50K in AIML filter(student, Fee > 50000 & Branch == "AIML") # c) Add Hostel_Charge = 20% of Fee student <- mutate(student, Hostel_Charge = Fee * 0.20)

String Manipulation — stringr Package

⚡ Likely 6M
💡 stringr = a package for working with text/strings in R. Think of it as "find, replace, split" operations on words and sentences.

Installation

install.packages("stringr") library("stringr")

Key Functions

s <- "Hello World Data Analytics" str_length(s) # 26 — count characters str_upper(s) # "HELLO WORLD DATA ANALYTICS" str_lower(s) # "hello world data analytics" str_sub(s, 1, 5) # "Hello" — substring str_replace(s, "World", "R") # "Hello R Data Analytics" str_detect(s, "Data") # TRUE — is "Data" in string? str_count(s, "a") # 4 — count occurrences of "a"

str_split() — Split a String ⭐ Most Asked!

Splits a string into pieces based on a separator character (like splitting a sentence into words).

# Split by space str_split("Hello World R", " ") # Output: [["Hello" "World" "R"]] # Split by comma str_split("Alice,Bob,Charlie", ",") # Output: [["Alice" "Bob" "Charlie"]] # Split a vector of strings names <- c("Alice Smith", "Bob Jones") str_split(names, " ") # Output: [[1]] "Alice" "Smith" [[2]] "Bob" "Jones"

Base R String Functions (also important)

nchar("Hello") # 5 — character count paste("A", "B", sep="-") # "A-B" strsplit("a,b,c", ",") # list: "a" "b" "c" grep("cat", c("cat","dog","catfish")) # 1 3 sub("o", "0", "Hello World") # "Hell0 World" (1st occurrence) gsub("o", "0", "Hello World") # "Hell0 W0rld" (all occurrences)
✅ Key Difference to Remember

sub() replaces FIRST match only. gsub() replaces ALL matches. str_split() = split by separator. str_detect() = check if pattern exists (TRUE/FALSE).

Data Import & Export in R

✅ Good to Know 6M
💡 Import = bringing data INTO R. Export = saving R data to a file. Most commonly used with CSV files.

📥 Importing Data

# Read CSV file df <- read.csv("data.csv") df <- read.csv("data.csv", header=TRUE, sep=",") # Read text file (tab-separated) df <- read.table("data.txt", header=TRUE, sep="\t") # Read Excel file (needs readxl package) library(readxl) df <- read_excel("data.xlsx")

# Write to CSV write.csv(df, "output.csv", row.names=FALSE) # write.table — more flexible write.table(df, file="output.txt", sep="\t", row.names=TRUE, col.names=TRUE) # write_tsv — tab-separated (needs readr package) library(readr) write_tsv(df, "output.tsv")

Descriptive Statistics & EDA in R

🔥 Very Likely 10M
💡 Descriptive Statistics = summarizing data with numbers (mean, median, etc.). EDA = exploring data visually to understand patterns before building models.

📊 The 5 Main Measures

x <- c(10, 20, 20, 30, 40, 50, 100) # 1. MEAN — Sum / Count = average mean(x) # 38.57 mean(x, trim=0.1, na.rm=TRUE) # trimmed mean (ignores extremes) # 2. MEDIAN — middle value (not affected by outliers) median(x) # 30 (sorted: 10,20,20,30,40,50,100) # 3. MODE — most frequent (R has no direct function) mode_val <- names(sort(table(x), decreasing=TRUE))[1] # mode_val = "20" # 4. VARIANCE — how spread out data is var(x) # 977.62 # 5. STANDARD DEVIATION — sqrt of variance (same unit as data) sd(x) # 31.27 # ALL AT ONCE summary(x) # Min, Q1, Median, Mean, Q3, Max fivenum(x) # Tukey 5-number summary quantile(x, c(0.25, 0.75)) # Q1 and Q3
✅ When to use Mean vs Median?

Use MEAN when data has no outliers (symmetric). Use MEDIAN when there are extreme values (skewed). In the example above, 100 is an outlier — median (30) is more representative than mean (38.57).

🔍 EDA — Exploratory Data Analysis

EDA is the process of exploring data before building any model. It helps you understand what's in the data.

Steps of EDA:

  • Step 1: Load & Inspect — head(), tail(), str(), dim(), names()
  • Step 2: Summary Statistics — summary(), mean(), sd()
  • Step 3: Check Missing Values — is.na(), sum(is.na(df))
  • Step 4: Visualize Distributions — hist(), boxplot()
  • Step 5: Find Correlations — cor(), pairs()
  • Step 6: Identify Outliers — boxplot shows outliers as dots
# EDA Example with mtcars dataset head(mtcars) # First 6 rows str(mtcars) # Structure: types of each column dim(mtcars) # 32 rows, 11 columns sum(is.na(mtcars)) # Count missing values (should be 0) summary(mtcars) # Statistics for all columns cor(mtcars) # Correlation matrix hist(mtcars$mpg) # Distribution of mpg boxplot(mpg ~ cyl, data=mtcars) # mpg grouped by cylinders

Building & Running R Packages

✅ Know Basics
💡 An R package is like an app — a collection of R functions, data, and documentation bundled together. Examples: dplyr, ggplot2, shiny are all packages.

How to Install & Use a Package

install.packages("dplyr") # Install from CRAN (internet needed) library("dplyr") # Load into current session require("dplyr") # Same as library but returns TRUE/FALSE installed.packages() # List all installed packages

Structure of a Package (DESCRIPTION file)

Every R package must have a DESCRIPTION file containing:

  • Package: Name of the package
  • Version: Version number (e.g., 1.0.0)
  • Author: Who made it
  • Depends: Other packages it needs
  • License: How it can be used

NAMESPACE File

Controls what functions are visible to users (exported) and what external functions are used (imported).

# In NAMESPACE file: export(myFunction) # Make myFunction available to users import(dplyr) # Import entire dplyr package importFrom(dplyr, filter) # Import only filter() from dplyr

Namespace Notation — Avoid Conflicts

# When two packages have same function name: dplyr::filter(df, Age > 20) # Use dplyr's filter stats::filter(x, filter) # Use stats' filter # :: means "from this package"

Create a Package in RStudio

File → New Project → New Directory → R Package

Uses the devtools package (by Hadley Wickham) which provides: load_all(), document(), test(), check(), build(), install()

Flexdashboard & R Shiny

⚡ Likely 6/10M
💡 Think of Flexdashboard as a "report template" and Shiny as the "engine that makes it interactive". Together they let you build live web dashboards using only R!

🎛️ What is Flexdashboard?

An R Markdown format that creates dashboard-style HTML pages. Can be static (just charts) or dynamic (with Shiny for interactivity).

Basic Setup (YAML Header)

--- title: "My Sales Dashboard" output: flexdashboard::flex_dashboard orientation: columns vertical_layout: fill ---

Layout Structure

# PAGE (use ===) Page 1 =================================== # COLUMN (use ---) Column 1 ----------------------------------- # PANEL (use ###) ### My Chart ```{r} hist(mtcars$mpg) ```

Special Components

valueBox()Display a KPI number with color/iconvalueBox(42, "Total Sales", icon="fa-cart")
gauge()Speedometer chartgauge(75, min=0, max=100)
datatable()Interactive searchable tableDT::datatable(df)
ggplotly()Interactive ggplot2 chartplotly::ggplotly(p)

🌐 What is R Shiny?

A framework to build interactive web apps in pure R. You define the UI (what user sees) and Server (what happens when they interact).

Basic Shiny App Structure

library(shiny) # Part 1: UI — what the user sees ui <- fluidPage( titlePanel("My Shiny App"), sidebarLayout( sidebarPanel( sliderInput("bins", "Number of bins:", min=1, max=50, value=30) ), mainPanel( plotOutput("distPlot") ) ) ) # Part 2: Server — what happens server <- function(input, output) { output$distPlot <- renderPlot({ x <- faithful[, 2] hist(x, breaks = input$bins, col = "blue") }) } # Part 3: Launch shinyApp(ui = ui, server = server)

Advantages of RShiny

  • No HTML/CSS/JavaScript knowledge needed
  • Real-time interactive charts and filters
  • Easy deployment to shinyapps.io
  • Integrates with ggplot2, plotly, DT
  • Runs in any web browser

Flexdashboard + Shiny Combined

--- title: "Interactive Dashboard" output: flexdashboard::flex_dashboard runtime: shiny # This line makes it interactive! --- ```{r} # Now you can use input$, output$, renderPlot() etc. sliderInput("n", "Sample size:", 10, 100, 50) renderPlot({ hist(rnorm(input$n)) }) ```
⚠️ Key Point for Exam

Flexdashboard alone = static HTML file. Add "runtime: shiny" in YAML = dynamic/interactive. Flexdashboard handles layout; Shiny handles reactivity. Know valueBox(), gauge(), and the basic ui + server structure.
